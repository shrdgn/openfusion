OpenAI-compatible compound-model proxy

Run a panel of models, stream one stronger answer.

openfusion fans each prompt out to a configurable panel of LLMs, asks a judge model to synthesize the best response, and returns it through the OpenAI chat completions API you already use.

How it works

Mixture-of-agents without a new client SDK.

01

Receive a normal chat request

The FastAPI service accepts OpenAI-shaped chat completions and passes through non-fusion model names unchanged.

02

Fan out to the panel

Configured panel members run in parallel with per-member timeouts and graceful degrade when an upstream fails.

03

Synthesize and stream

A judge model reads the panel outputs, resolves contradictions, and streams one OpenAI-compatible answer back to the client.

Quick start

Run openfusion locally.

Bring an upstream OpenAI-compatible provider key, then point your client at localhost.

pip install -e .
cp openfusion.yaml.example openfusion.yaml
export OPENROUTER_API_KEY=your-key-here
openfusion --host 0.0.0.0 --port 8000
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="local-dev",
)

stream = client.chat.completions.create(
    model="openfusion",
    messages=[{"role": "user", "content": "Explain MoA."}],
    stream=True,
)

Safety model

Built to keep provider access server-side.

Provider keys stay in config

Client bearer tokens are gateway credentials; upstream API keys come from env-backed YAML.

Cost controls are explicit

Token ceilings cap pass-through, panel, and judge requests before credit burn surprises you.

Logs avoid prompt data

Operational logs focus on metadata, latency, status, usage, and cost rather than prompt text.

Hosted version comes later

A hosted product should add account isolation, rate limits, budget controls, and abuse handling before accepting real user traffic.

Project path

Open source first, hosted when the operational boundary is real.

This landing page is served by the package today so local and hosted deployments have a clear root. If the project picks up, the marketing site or hosted app can move out without changing the proxy API surface.
